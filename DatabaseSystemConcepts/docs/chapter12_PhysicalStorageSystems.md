# Chapter 12: Physical Storage Systems

## 개요

데이터베이스의 물리적 저장 계층을 다룬다. 저장 매체의 계층 구조(캐시→RAM→플래시→자기 디스크→광학→테이프), 자기 디스크의 물리적 특성과 성능 지표, 플래시 메모리(SSD) 동작 원리, 그리고 신뢰성·성능을 동시에 확보하는 RAID 기법을 상세히 설명한다.

---

## 12.1 물리적 저장 매체 개요

### 저장 계층 구조

```
속도 (빠름)                                        비용/바이트 (비쌈)
       ┌──────────────────────────────────────────────────────┐
       │                   캐시 (Cache)                        │
       │           CPU 내장, HW 관리, 수 MB                    │
       ├──────────────────────────────────────────────────────┤
       │                 주 메모리 (Main Memory)               │
       │         수 GB ~ 수 TB, 휘발성(Volatile)               │
       ├──────────────────────────────────────────────────────┤
       │              플래시 메모리 (Flash / SSD)              │
       │        비휘발성, 자기 디스크보다 빠름, 비쌈           │
       ├──────────────────────────────────────────────────────┤
       │             자기 디스크 (Magnetic Disk / HDD)         │
       │       비휘발성, 대용량, 저비용, 랜덤 접근 느림        │
       ├──────────────────────────────────────────────────────┤
       │              광학 저장 (Optical / DVD)                │
       │         주로 백업·배포용, 비교적 느린 접근            │
       ├──────────────────────────────────────────────────────┤
       │              자기 테이프 (Magnetic Tape)              │
       │        순차 접근만 가능, 아카이브·백업 용도           │
       └──────────────────────────────────────────────────────┘
속도 (느림)                                        비용/바이트 (쌈)
```

| 계층 | 휘발성 | 접근 방식 | 용도 |
|------|--------|-----------|------|
| 캐시 | 휘발 | 직접 | CPU 가속 |
| 주 메모리 | 휘발 | 직접 | 연산 공간 |
| 플래시/SSD | **비휘발** | 직접 | 영속 저장 |
| 자기 디스크 | **비휘발** | 직접 | 영속 저장 |
| 광학 | **비휘발** | 직접 | 백업 |
| 테이프 | **비휘발** | **순차** | 아카이브 |

- **1차 저장(Primary)**: 캐시 + 주 메모리
- **2차 저장(Secondary / Online)**: 플래시 + 자기 디스크
- **3차 저장(Tertiary / Offline)**: 광학 + 테이프

## 12.2 저장 인터페이스

| 인터페이스 | 최대 속도 | 특징 |
|------------|-----------|------|
| SATA-3 | ~600 MB/s | 소비자용 HDD/SSD |
| SAS v3 | ~1.2 GB/s | 서버용, 엔터프라이즈 |
| NVMe/PCIe | ~3+ GB/s | SSD 전용 고속 인터페이스 |

**SAN (Storage Area Network)**: 대량의 디스크를 고속 네트워크(iSCSI, Fibre Channel, InfiniBand)로 서버에 연결. RAID로 구성된 대용량 가상 단일 디스크 제공.

**NAS (Network Attached Storage)**: SAN과 유사하나 파일 시스템 인터페이스 제공 (NFS, CIFS).

## 12.3 자기 디스크 (Magnetic Disk)

### 12.3.1 물리적 구조

```
          스핀들(Spindle)
             │
    ┌────────┴────────┐
    │    플래터(Platter)│   ← 여러 장
    │  ┌──────────────┤
    │  │  트랙(Track)  │   ← 동심원 형태
    │  │  섹터(Sector) │   ← 트랙을 분할한 최소 I/O 단위 (보통 512B)
    │  └──────────────┤
    └─────────────────┘
           ↑
   디스크 암(Disk Arm) ─── 읽기/쓰기 헤드
   (모든 플래터 헤드가 동시에 이동)

실린더(Cylinder) = 모든 플래터의 동일 트랙 번호 집합
```

- **헤드 크래시(Head Crash)**: 헤드가 디스크 표면에 접촉 → 데이터 손상
- **체크섬(Checksum)**: 섹터별 기록, 읽기 시 검증으로 오류 감지
- **불량 섹터 재매핑**: 컨트롤러가 예비 섹터로 투명하게 대체

### 12.3.2 성능 지표

```
전체 접근 시간 = 탐색 시간 + 회전 지연 시간
                   seek time   rotational latency

데이터 전송률: 순차 읽기 기준 50~200 MB/s (최대), 내부 트랙은 30 MB/s 수준
IOPS (4KB 블록): 50~200 IOPS (랜덤 접근)
MTTF: 500,000~1,200,000 시간 (실제 수명 약 5년)
```

| 지표 | 일반적 값 | 설명 |
|------|-----------|------|
| 탐색 시간(Seek) | 2~20 ms | 암 이동 시간, 평균 ≈ 최대의 1/2 |
| 회전 지연(Rotational Latency) | 2~5.5 ms | 평균 = 1/2 회전 시간 |
| 총 접근 시간 | 5~20 ms | Seek + Latency |
| 데이터 전송률 | 50~200 MB/s | 순차 최대값 |

**랜덤 vs 순차**:
- 순차: 첫 블록 탐색 후 인접 트랙 연속 읽기 → 높은 처리량
- 랜덤: 요청마다 탐색 → IOPS 제한 (4KB 블록 기준 50~200)

## 12.4 플래시 메모리 (Flash Memory / SSD)

### NAND 플래시 구조

```
플래시 주소 계층:
Page (4KB) → Erase Block (256KB~1MB, ~128~256 pages) → Chip

읽기: 페이지 단위 (4KB), 20~100 μs
쓰기: 페이지 단위 (4KB), ~100 μs  ← 단, 사전 소거 필요
소거: 이레이스 블록 단위, 2~5 ms   ← 느림!
수명: 페이지당 소거 횟수 100,000~1,000,000회
```

### 플래시 변환 계층 (FTL: Flash Translation Layer)

**핵심 문제**: NAND 플래시는 in-place 덮어쓰기 불가 → 논리-물리 페이지 주소 매핑 필요.

```
논리 페이지 번호 → [FTL 변환 테이블] → 물리 페이지 번호

업데이트 흐름:
1. 새 논리 페이지 → 빈 물리 페이지에 기록
2. 기존 물리 페이지 = "삭제됨(deleted)" 표시
3. 가비지 컬렉션: 삭제된 페이지가 많은 블록을 소거
```

**웨어 레벨링(Wear Leveling)**: 물리 페이지 소거 횟수를 균등화하여 수명 연장. 자주 업데이트되는 데이터(hot data)는 소거 횟수가 적은 페이지로, 거의 변경 없는 데이터(cold data)는 소거 횟수가 많은 페이지로 배정.

### SSD 성능 지표

| 지표 | SATA SSD | NVMe PCIe SSD |
|------|----------|---------------|
| 순차 읽기 | ~500 MB/s | ~3+ GB/s |
| 순차 쓰기 | ~500 MB/s | ~2~3 GB/s |
| 랜덤 읽기 (QD-1) | ~10K IOPS | ~50K IOPS |
| 랜덤 읽기 (QD-32) | ~100K IOPS | ~350K+ IOPS |
| 읽기 지연 | 20~100 μs | 20~100 μs |

**스토리지 클래스 메모리(Storage Class Memory)**: 3D-XPoint (Intel Optane) 등 바이트 단위 접근 가능한 비휘발성 메모리. RAM과 SSD의 중간 비용·지연·용량.

## 12.5 RAID

**RAID (Redundant Arrays of Independent Disks)**: 다중 디스크를 통해 성능·신뢰성·관리성을 동시에 개선.

### 12.5.1 패리티(Parity)를 이용한 신뢰성 향상

**미러링(Mirroring)**: 모든 디스크를 2중 복제. 비용이 가장 비싸나 복구가 가장 빠름.

```
MTTF_mirrored ≈ MTTF_single² / (2 × MTTR)
단일 디스크 MTTF = 100,000h, MTTR = 10h이면:
  → 100,000² / 20 = 500,000,000h ≈ 57,000년
```

**패리티 블록**: XOR 기반으로 하나의 디스크 장애 복구 가능.
```
parity_bit[i] = d1[i] XOR d2[i] XOR ... XOR dN[i]
장애 디스크 복구: 나머지 디스크 + 패리티의 XOR
```

### 12.5.2 RAID 레벨

```
RAID 0: [D0][D1][D2][D3]          스트라이핑만, 중복 없음 → 성능↑, 신뢰성 없음
RAID 1: [D0][D1][D2][D3]
        [C0][C1][C2][C3]          미러링 → 신뢰성 최고, 비용 2배
RAID 5: [D0][D1][D2][P0]
        [D3][D4][P1][D5]          분산 패리티 → 읽기 성능↑, 쓰기는 패리티 계산 필요
        [D6][P2][D7][D8]
        [P3][D9][D10][D11]
RAID 6: [D0][D1][D2][P][Q]        P+Q 이중 오류 정정(Reed-Solomon) → 2개 동시 디스크 장애 허용
```

| RAID 레벨 | 디스크 오버헤드 | 쓰기 성능 | 읽기 성능 | 허용 장애 | 주 용도 |
|-----------|----------------|-----------|-----------|-----------|---------|
| 0 | 없음 | 최고 | 최고 | 0개 | 임시 데이터 |
| 1 | 100% | 최고 | 높음 | N/2개 | 로그 파일, 고I/O |
| 5 | 1/N | 중간 (4I/O/write) | 높음 | 1개 | 읽기 위주 워크로드 |
| 6 | 2/N | 낮음 (6I/O/write) | 높음 | 2개 | 고가용성 |

**RAID 5 랜덤 쓰기 비용**:
```
블록 1개 랜덤 쓰기 = 2 읽기(구 데이터 + 구 패리티) + 2 쓰기(새 데이터 + 새 패리티)
                   = 4 I/O 연산
```

### 12.5.3 RAID 레벨 선택 기준

| 상황 | 권장 레벨 |
|------|-----------|
| 무결성 불필요, 최고 성능 필요 | RAID 0 |
| 랜덤 쓰기 많음, 고가용성 | RAID 1 |
| 읽기 많음, 저비용, 1개 장애 허용 | RAID 5 |
| 최고 가용성, 2개 동시 장애 허용 | RAID 6 |

### 12.5.4 하드웨어 RAID 고려 사항

- **소프트웨어 RAID**: OS 수준 구현. 파워 장애 후 재동기화(resync) 과정 필요.
- **하드웨어 RAID**: NVRAM 기반 비휘발성 쓰기 버퍼 포함. 파워 장애 시에도 일관성 보장.
- **Hot Swapping**: 시스템 종료 없이 고장 디스크 교체. MTTR 단축.
- **스크러빙(Scrubbing)**: 유휴 시간에 전체 디스크 스캔 → 잠재 장애(bit rot) 조기 탐지.

## 12.6 디스크 블록 접근 최적화

### 접근 패턴

| 패턴 | 특징 | 성능 |
|------|------|------|
| **순차 접근(Sequential)** | 연속 블록 번호 요청 | 높은 처리량, 적은 탐색 |
| **랜덤 접근(Random)** | 임의 블록 번호 요청 | 낮은 IOPS, 매 요청마다 탐색 |

### 최적화 기법

1. **버퍼링(Buffering)**: 읽은 블록을 메모리에 캐시하여 재접근 시 디스크 I/O 회피.

2. **선독(Read-Ahead)**: 현재 블록 주변 연속 블록을 미리 읽어 순차 접근 지연 최소화.

3. **디스크 암 스케줄링(Disk-Arm Scheduling)**:
   - **엘리베이터 알고리즘(Elevator Algorithm)**: 암이 한 방향으로 이동하면서 요청된 트랙 순서대로 처리, 끝에 도달하면 방향 반전.
   ```
   암 이동 방향 →  0  10  30  47  55  (대기 중인 트랙)
   처리 순서: 10 → 30 → 47 → 55 → (방향 반전)
   ```

4. **파일 구성(File Organization)**: 순차 접근 예상 파일은 연속 블록에 할당. 익스텐트(Extent) 단위 할당.

5. **비휘발성 쓰기 버퍼(NVRAM Write Buffer)**: 쓰기 요청을 NVRAM에 즉시 기록 후 응답. 디스크 쓰기는 나중에 최적화된 순서로 처리.

6. **로그 디스크(Log Disk)**: 전용 순차 디스크에 쓰기 순서 기록 → 저널링 파일 시스템(Journaling FS). 크래시 복구 시 로그 디스크를 재생하여 일관성 복원.

## 12.7 핵심 요약

```
저장 계층 선택 가이드

질문: 데이터가 얼마나 자주 접근되는가?
  ↓
항상 최신 실시간 접근 필요 → 주 메모리 / SSD
  ↓
빈번한 랜덤 접근 필요 → SSD (NVMe)
  ↓
대용량 + 저비용 허용 → HDD
  ↓
장기 보관 / 백업 → 테이프
```

| 개념 | 핵심 포인트 |
|------|------------|
| 저장 계층 | 속도↑ = 비용↑ = 용량↓, 비휘발성은 플래시부터 |
| 자기 디스크 | Seek + Rotational Latency = 5~20ms 접근 시간 |
| SSD/NAND | 읽기 빠름, 쓰기는 소거 필요 → FTL이 투명하게 관리 |
| Wear Leveling | 쓰기 균등 분산으로 SSD 수명 연장 |
| RAID 0 | 스트라이핑만, 신뢰성 없음 |
| RAID 1 | 미러링, 최고 신뢰성, 2배 비용 |
| RAID 5 | 분산 패리티, 1개 장애 허용, 랜덤 쓰기 4 I/O |
| RAID 6 | P+Q 이중 패리티, 2개 동시 장애 허용 |
| 엘리베이터 알고리즘 | 디스크 암 이동 방향 유지로 탐색 횟수 최소화 |
| NVRAM 버퍼 | 비휘발성 쓰기 버퍼로 쓰기 지연 최소화 |
