# OS / Linux 핵심 정리

---

## 1. 프로세스 vs 스레드

| 항목 | 프로세스 | 스레드 |
|------|----------|--------|
| 메모리 공간 | 독립적 (별도 주소 공간) | 공유 (힙, 데이터 영역 공유) |
| 생성 비용 | 높음 | 낮음 |
| 통신 방식 | IPC (파이프, 소켓, 공유메모리) | 공유 메모리 직접 접근 |
| 장애 격리 | 한 프로세스 죽어도 다른 프로세스 무관 | 한 스레드 크래시 시 전체 프로세스 영향 |

**컨텍스트 스위칭**: CPU가 실행 중인 프로세스/스레드를 교체할 때 레지스터, PC, 스택 포인터 등 상태를 저장/복원하는 작업. 프로세스 간 전환이 스레드 간 전환보다 비용이 큼.

## 2. 메모리 구조

```
높은 주소
┌─────────────┐
│    Stack    │ ← 지역변수, 함수 호출 프레임 (자동 할당/해제)
├─────────────┤
│     ↓       │
│             │
│     ↑       │
├─────────────┤
│    Heap     │ ← 동적 할당 (malloc/new), 개발자가 직접 관리
├─────────────┤
│    BSS      │ ← 초기화되지 않은 전역/정적 변수
├─────────────┤
│    Data     │ ← 초기화된 전역/정적 변수
├─────────────┤
│    Text     │ ← 실행 코드 (읽기 전용)
낮은 주소
```

## 3. 가상 메모리 / 페이징

- **가상 메모리**: 각 프로세스에 독립적인 주소 공간 제공. 실제 물리 메모리보다 큰 공간 사용 가능
- **페이지**: 가상 메모리를 고정 크기(보통 4KB)로 나눈 단위
- **페이지 테이블**: 가상 주소 → 물리 주소 매핑 테이블
- **페이지 폴트**: 요청한 페이지가 물리 메모리에 없을 때 발생 → OS가 디스크에서 로드
- **TLB(Translation Lookaside Buffer)**: 페이지 테이블 캐시, 주소 변환 속도 향상

## 4. 동기화 기법

| 기법 | 설명 | 특징 |
|------|------|------|
| **Mutex** | 상호 배제 잠금. 소유권 개념 있음 | 잠금 획득한 스레드만 해제 가능 |
| **Semaphore** | 카운터 기반 잠금. N개 동시 접근 허용 | 소유권 없음, 다른 스레드가 해제 가능 |
| **Spinlock** | 잠금 획득까지 CPU를 계속 점유(busy-wait) | 대기 시간이 매우 짧을 때 유리 |
| **RWLock** | 읽기는 동시 허용, 쓰기는 독점 | 읽기 많은 환경에서 유리 |
| **Atomic** | CPU 명령어 수준의 원자적 연산 | 락 없이 간단한 연산에 사용 |

**데드락 조건 (4가지 모두 충족 시 발생)**
1. 상호 배제 (Mutual Exclusion)
2. 점유 대기 (Hold and Wait)
3. 비선점 (No Preemption)
4. 순환 대기 (Circular Wait)

## 5. 스케줄링 알고리즘

- **FCFS**: 먼저 온 순서대로 처리. 단순하지만 호위 효과(Convoy Effect) 문제
- **SJF**: 실행 시간이 짧은 것 먼저. 기아(Starvation) 문제
- **Round Robin**: 시간 할당량(time quantum) 단위로 순환. 응답성 좋음
- **Priority Scheduling**: 우선순위 기반. 기아 문제 → 에이징(Aging)으로 해결
- **CFS (Linux)**: 완전 공정 스케줄러. 각 프로세스의 가상 실행 시간을 추적

## 6. Linux 핵심 명령어

```bash
# 프로세스
ps aux                    # 전체 프로세스 목록
top / htop                # 실시간 프로세스 모니터링
kill -9 <pid>             # 강제 종료
pgrep <name>              # 이름으로 PID 검색
lsof -p <pid>             # 프로세스가 열고 있는 파일 목록
strace -p <pid>           # 시스템 콜 추적

# 네트워크
ss -tlnp                  # 리스닝 포트 확인 (netstat 대체)
netstat -anp              # 연결 상태 확인
tcpdump -i eth0 port 80   # 패킷 캡처
iptables -L               # 방화벽 규칙 확인

# 파일 시스템
df -h                     # 디스크 사용량
du -sh *                  # 디렉토리별 용량
find / -name "*.log" -mtime +7  # 7일 이상 된 로그 파일 찾기
tail -f /var/log/syslog   # 실시간 로그 확인

# 시스템 리소스
free -h                   # 메모리 사용량
vmstat 1                  # 시스템 통계 1초 간격
iostat -x 1               # I/O 통계
ulimit -n                 # 파일 디스크립터 제한 확인
```

## 7. 파일 디스크립터와 I/O 모델

- **파일 디스크립터**: 열린 파일/소켓을 나타내는 정수. 기본값 0(stdin), 1(stdout), 2(stderr)
- **ulimit -n**: 프로세스당 열 수 있는 최대 fd 수 (기본 1024, 서버는 보통 65535 이상으로 높임)

**I/O 모델 비교**

| 모델 | 설명 |
|------|------|
| Blocking I/O | 작업 완료될 때까지 블록. 가장 단순 |
| Non-blocking I/O | 즉시 반환, 완료 여부 폴링 필요 |
| I/O Multiplexing | `select`/`poll`/`epoll` 로 여러 fd 감시 |
| Async I/O (AIO) | 커널이 완료 후 콜백 통지 |

**epoll vs select**
- `select`: fd 최대 1024개 제한, O(n) 스캔
- `epoll`: fd 제한 없음, O(1) 이벤트 감지. Linux 고성능 서버의 표준

## 8. 시그널 (Signal)

| 시그널 | 번호 | 의미 |
|--------|------|------|
| SIGTERM | 15 | 정상 종료 요청 (처리 가능) |
| SIGKILL | 9 | 강제 종료 (처리 불가) |
| SIGINT | 2 | Ctrl+C |
| SIGHUP | 1 | 터미널 끊김, 설정 재로드에 활용 |
| SIGUSR1/2 | 10/12 | 사용자 정의 시그널 |

## 9. 부팅 과정 (Linux)

1. BIOS/UEFI → POST 수행
2. 부트로더 (GRUB) 실행
3. 커널 로드 및 초기화
4. `init` / `systemd` (PID 1) 실행
5. 서비스 및 데몬 시작
6. 로그인 프롬프트

**systemctl** 주요 명령
```bash
systemctl start/stop/restart nginx
systemctl enable nginx      # 부팅 시 자동 시작
systemctl status nginx
journalctl -u nginx -f      # 서비스 로그
```

## 10. 자주 나오는 면접 질문

**Q. 좀비 프로세스란?**
자식 프로세스가 종료됐지만 부모가 `wait()`를 호출하지 않아 프로세스 테이블에 항목이 남아있는 상태. PID를 점유하지만 실제 리소스는 없음.

**Q. 고아 프로세스란?**
부모 프로세스가 먼저 종료된 자식 프로세스. init(PID 1)이 자동으로 입양해 `wait()` 처리.

**Q. Copy-on-Write(COW)란?**
`fork()` 시 부모/자식이 동일한 물리 메모리 페이지를 공유하다가, 실제로 쓰기가 발생할 때만 별도 복사본을 만드는 최적화 기법.

**Q. Linux에서 파일은 모든 것이다(Everything is a file)?**
소켓, 파이프, 디바이스, 디렉토리 등 모두 파일 인터페이스(open/read/write/close)로 추상화됨.
